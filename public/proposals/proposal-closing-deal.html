Product Proposal · 요기요 할인랭킹 진화

마감딜

팔리지 않으면 버려야 하는 메뉴가 있습니다. 사장님은 이미 20~30% 할인할 준비가 돼 있습니다.
입찰 경쟁이 아닌 시간 기반 마감 할인으로 구조를 바꿉니다.

Problem — 기존 할인랭킹의 3가지 구조적 한계
1
지방 이미지 훼손
지방은 가게 수가 적어 200원만 할인해도 할인랭킹 1위. 할인 경쟁처럼 보이지 않아 소비자 경험 저하 — 할인랭킹의 무게감 자체가 무너짐.
소비자 경험 저하
2
할인율이 낮음
광고비 지출 방향으로 흘러 실제 체감 할인이 없음. 소비자 입장에서 배민·쿠팡이츠 대비 차별화 불가.
소비자 유인 부족
3
상위 고착화
오래 광고한 업장이 계속 상위 → 신규 입점 업장 노출 기회 없음. 플랫폼 내 불평등 심화.
신규 업장 불이익
🕐
핵심 논리 — "버리느니 파는 게 낫다"

반찬·밀키트·베이커리·샐러드는 당일 마감 재고가 존재합니다.
폐기 비용을 감수하느니 20~40% 할인해서 파는 게 낫습니다.
사장님이 자발적으로 할인을 선택하는 유일한 상황입니다. 요기요는 그 타이밍에 노출만 해주면 됩니다.

Solution — 할인랭킹 vs 마감딜
구분 기존 할인랭킹 마감딜 NEW
노출 기준 광고비 입찰 경쟁 마감 시간 임박 순 자동
할인 주체 사장님 입찰 금액 설정 (광고비 기반) 사장님 마감 할인율 직접 설정 (재고 처리 기반)
할인율 광고비 비용 감안 시 소비자 체감 낮음 폐기 회피 동기 → 사장님이 자율 결정
소비자 경험 항상 비슷한 업장 노출, 신선도 낮음 오늘 한정 특가 → 긴급성·희소성 부여
사장님 부담 지속적 광고비 지출 필요 팔릴 때만 % 설정, 수수료 없음

✓
목적이 다름
할인랭킹 = 상시 노출 경쟁 (광고). 마감특가 = 마감 재고 처리 (손실 최소화). 사장님 동기가 근본적으로 다릅니다.
대체제 아님
✓
시간대가 분리됨
할인랭킹 = 24시간 상시. 마감특가 = 영업 마감 임박 특정 시간. 소비자도 서로 다른 맥락에서 소비합니다.
보완재
✓
지방 이미지 문제 해소
마감특가가 생기면 "진짜 할인"은 마감특가 탭으로, 할인랭킹은 상시 노출 탭으로 역할 분리. 지방의 200원 1위 문제가 자연스럽게 희석됩니다.
할인랭킹 이미지 보강
Why Now — 요기요에게 이게 왜 중요한가
지방 역전
지방에서 오히려 더 잘 작동
입찰 경쟁 없이 마감 재고 기반 자동 노출. 수도권 대형 업장이 없는 지방에서 진짜 할인이 만들어지고, 소비자 경험이 더 좋아짐.
비용 제로
플랫폼 보조금 불필요
사장님이 자발적으로 할인 설정. 요기요가 쿠폰 비용을 부담하지 않아도 20~40% 할인이 만들어짐. 지속 가능한 구조.
차별화
배민·쿠팡이츠에 없는 포맷
배민 타임딜은 일부 운영, 구조적 마감딜 탭은 3사 모두 없음. 먼저 만들면 "요기요=마감 할인" 이미지 선점 가능.
MVP 범위 — 무엇부터 만드나
Phase 1 · 3주

마감딜 탭 + 등록 + 배지

  • 사장님센터: 마감 시간·할인율·잔여 수량 설정
  • 소비자 앱: "마감딜" 탭 신설, 마감 임박순 자동 정렬
  • 업장 리스트 배지 표시 (마감딜 진행 중)
  • 최소 할인율 기준 적용 (반찬·베이커리 15%, 밀키트 20%+)
  • 반찬·베이커리·밀키트 카테고리 우선 온보딩
Phase 1.5 · +2주

정기예약 등록

  • 사장님센터: 요일·시간대 반복 등록 (매일 19시 30% 등)
  • 딜 준비 부담 최소화 → 습관적 사용 유도
  • 사장님: 딜별 매출·폐기율 감소 리포트
  • 수수료 구간 연동 — 추가 주문 건수 → 수수료 절감 표시
Phase 2 · 법무 검토 후

단골 알림 + AI 연동

  • 소비자 푸시 알림 "단골 카테고리 마감딜 시작" (옵트인)
  • 개인정보·마케팅 동의 처리 법무 검토 필요
  • AI 어드바이저 연동 — "지금 등록하면 주문 예상 +N건"
  • 카테고리 확장 (국·탕·찌개, 샐러드 추가)
할인랭킹과의 차이
할인랭킹 = 노출 경쟁 (광고비)
마감딜 = 재고 소진 (마감 시점 노출)
