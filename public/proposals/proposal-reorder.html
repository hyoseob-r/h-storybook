재주문 숏컷 — 버티컬 특화 전략

바니프레소가 커피 하나로 만든 것을 요기요는 모든 카테고리에서 만들 수 있습니다.
"자주 주문한 이 메뉴 어때요?"를 컨텍스트 인식 재주문으로 교체합니다.

진단 — 왜 지금 재주문 UI가 약한가
현재 구조
"자주 주문한 이 메뉴 어때요?"
시간·위치·패턴 무관하게 단일 레이어. 저녁 11시에 아침 커피 추천, 주말에 평일 도시락 노출. 카테고리 혼합 → 맥락 없는 카드 나열 → 스크롤 없이 지나침.
맥락 없음 · 버티컬 없음 · 습관 형성 없음
→
제안 구조
컨텍스트 인식 재주문
앱을 여는 시간·요일·위치·주기 데이터를 조합해 지금 이 순간에 맞는 버티컬 하나만 노출. 바니프레소처럼 "오늘도 아메리카노?" 수준의 습관 루프를 모든 카테고리에서 구현.
시간 기반 · 주기 감지 · 루트 연계 · 패턴 학습
레퍼런스 — 바니프레소가 단일 브랜드 앱으로 해낸 것
⏰
시간 컨텍스트
아침 출근 루틴에 완전 최적화. 7~9시에 앱을 열면 바로 "오늘도 [단골 메뉴]?"가 전면 노출. 다른 선택지 없음.
→ 결정 피로 제거
📍
픽업 예약
도착 시간 역산 → "15분 후 픽업"으로 주문. 배달 대기 없이 가게 앞에서 수령. 출퇴근 동선에 완전 내재화.
→ 배달 의존 탈피
🔁
습관 루프
첫 주문 → 편의성 체험 → 다음 날 또 → 3주 후 무의식. 앱이 습관의 트리거가 됨. 경쟁 앱 진입 차단.
→ 락인 없는 락인
요기요의 어드밴티지: 바니프레소는 커피 하나. 요기요는 커피·반찬·야식·마트·밀키트 — 버티컬마다 각각의 바니프레소를 만들 수 있습니다.
버티컬 스윔레인 — 6개 컨텍스트 레이어
☀️ 아침 루틴
카페 버티컬 — 바니프레소 모드
트리거: 평일 오전 7~9시 앱 오픈
단골 카페 메뉴를 전면 카드로 노출. 픽업 시간 선택(10/20/30분 후)만 누르면 주문 완료. 3주간 패턴이 쌓이면 앱 열자마자 "오늘도 [카페명]?" 배너로 선점.
🗓️ 주기 감지
반찬·밀키트 버티컬 — 소진 예측
트리거: 마지막 주문 후 학습된 주기 도달
구매 주기 자동 학습(7일/10일/14일). "지난번 [반찬가게] 주문한지 12일 됐어요" 알림 발송. 푸시 → 앱 진입 → 원탭 재주문. 소진 타이밍에 딱 맞게 도착.
🍱 점심 단골
편의식 버티컬 — 점심 루틴
트리거: 평일 11:30~12:30 + 자주 주문한 위치
사무실·학교 근처 단골 도시락·분식 숏컷. "지금 주문하면 12:30 도착" 도착 시간을 전면에. 같은 메뉴를 5회 이상 주문 시 자동으로 "오늘의 점심" 레이블 부여.
🗺️ 퇴근 루트
픽업 루트 버티컬 — 동선 내재화
트리거: 퇴근 시간대 + 설정된 귀가 루트
퇴근 루트 주요 지점 등록 → 경로상 단골 매장 미리 주문. "지금 지하철 탑승하셨나요? [강남역 단골집] 미리 주문하면 도착 시 픽업 가능." 바니프레소의 핵심을 요기요 전체 카테고리로 확장.
📦 주간 장보기
마트·생필품 버티컬 — 번들 재주문
트리거: 매주 동일 요일 + 마트/장보기 카테고리 패턴
지난번 장바구니를 통째로 재주문. 품목 조정만 하면 주문 완료. "지난 일요일 [마트명] 장바구니 그대로?" → 확인 화면에서 빠른 수정 가능. 월 1~2회 주기 패턴에 최적화.
🌙 야식 패턴
야식 버티컬 — 요일·감성 기반
트리거: 금·토 오후 9시 이후
금/토 심야 단골 치킨·피자·분식 숏컷. "지난 금요일 그 집, 오늘도?" 형태로 감성 터치. 날씨 연동(비 오는 날 + 야식 패턴) 보너스 노출로 전환율 극대화.

컨텍스트 홈의 3가지 레이어
1
컨텍스트 배너 (최상단)
지금 이 순간 가장 관련성 높은 버티컬 1개를 전면 배너로 점유. 시간·위치·패턴 조합으로 결정. 패턴이 없으면 배너 없음.
2
주기 알림 레인
주기 임박한 가게를 최대 3개 카드로. 소진 예측 기반이라 배달비 할인 메시지 없이도 자연스럽게 전환 유도.
3
시간대 단골 레인
점심·저녁 시간대엔 해당 버티컬 단골만 노출. 기존 "자주 주문한 메뉴"를 대체 — 맥락 없는 카드 나열 제거.
⚡ 픽업 예약 통합 시 킬러 기능
배달만 지원하는 현재 구조에서, 픽업 예약 지원 시 바니프레소 모델을 요기요 전 카테고리로 확장 가능. 카페 버티컬만으로도 아침 DAU 대폭 상승 기대.
패턴 학습 엔진 — 3단계 개인화
1
관찰기 (1~3주)
주문 시간대, 요일, 카테고리, 위치를 수집. 추천 없음. 최소 5회 주문 데이터 확보 후 다음 단계로.
침묵 수집
2
감지기 (4~6주)
패턴 신뢰도 70% 이상일 때만 첫 추천 노출. 거부 시 신뢰도 리셋. 클릭 시 가중치 +1.5배 부여.
조심스러운 첫 제안
3
정교화 (7주~)
CTR·전환율로 추천 품질 자동 조정. 버티컬 간 충돌 시 최고 신뢰도 버티컬 우선. 계절·날씨 보정 추가.

파트너 AI 시너지 — 소비자 데이터 → 사장님 도구
소비자 앱 (재주문 숏컷)
쌓이는 행동 데이터
  • 재주문 주기 (7일/14일 패턴)
  • 픽업 vs. 배달 선호도
  • 이탈 시점 (언제 안 왔는지)
  • 버티컬 전환 패턴 (커피→반찬→야식)
  • 가격 탄력성 (할인 유무에 따른 반응)
파트너 AI (사장님 도구)
사장님이 받는 인사이트
  • "이 손님 매주 화요일 오세요 → 화요일 쿠폰 자동 발송"
  • "이 손님 2주째 안 오셨어요 → 재방문 유도 알림"
  • "이 시간대 픽업 주문 집중 → 픽업 운영 시간 최적화"
  • "단골 손님 주기 임박 → 선제 할인 제안 타이밍"
플라이휠: 재주문 숏컷이 쌓을수록 파트너 AI의 추천 정밀도가 올라갑니다. 사장님이 적절한 시점에 쿠폰을 보내면 소비자가 다시 오고, 소비자가 오면 재주문 데이터가 다시 쌓입니다. 버티컬이 많을수록, 사용 기간이 길수록 격차는 벌어집니다.
